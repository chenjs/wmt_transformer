# 改进后的训练配置建议

基于架构改进和之前的诊断结果，以下是为改进后的Transformer模型推荐训练参数。

## 架构改进总结

1. **Pre-norm架构**：将LayerNorm移到残差连接之前，改善梯度流，训练更稳定
2. **更好的初始化**：
   - Embedding层：正态分布N(0, 0.02²)
   - 线性层：Xavier均匀初始化
   - LayerNorm：权重=1，偏置=0

## 训练参数建议

### 配置文件修改 (`src/config.py`)

```python
# 在Config类中修改以下参数：

# 训练参数
batch_size: int = 32  # 增加到32以稳定训练
learning_rate: float = 1e-3  # 从5e-4提高到1e-3
warmup_steps: int = 4000  # 恢复标准warmup
max_steps: int = 100000    # 保持足够训练步数
label_smoothing: float = 0.0  # 暂时禁用，词汇量大时0.1过于激进
clip_grad: float = 5.0     # 从1.0放宽到5.0，防止过度裁剪

# 数据
max_train_samples: int = 200000  # 增加到200k样本
```

### 训练脚本调整 (`scripts/train.py`)

无需修改，但建议添加以下监控：

1. **定期验证**：每5000步在验证集上评估
2. **学习率调度**：确保warmup_steps > 0
3. **梯度监控**：记录梯度范数

### 推荐训练命令


## 预期改进

### 训练稳定性
1. **梯度流改善**：Pre-norm架构减少梯度消失问题
2. **学习率适应性**：1e-3学习率配合warmup，加速早期训练
3. **梯度裁剪放宽**：5.0的阈值允许更大梯度更新

### 模型性能
1. **编码器输出多样性**：不同输入应产生显著不同编码
2. **解码器预测多样性**：不再重复输出相同token
3. **损失下降曲线**：应稳定下降而非平缓

## 监控指标

### 关键检查点
1. **前1000步**：损失应显著下降（从~7.2降至~6.0以下）
2. **5000步**：不同输入应产生不同翻译输出
3. **20000步**：验证集BLEU分数应开始上升

### 警告信号
1. **损失不下降**：检查梯度范数、学习率
2. **输出重复**：检查encoder输出多样性
3. **梯度爆炸**：减小学习率或增加梯度裁剪

## 故障排除

### 问题1：损失下降缓慢
**解决方案**：
- 增加学习率到2e-3
- 禁用标签平滑（已设置label_smoothing=0.0）
- 检查数据预处理是否正确

### 问题2：输出仍然重复
**解决方案**：
- 验证tokenizer是否正确使用（已修复）
- 检查encoder输出标准差应>0.5
- 确保batch_size足够大（≥32）

### 问题3：梯度消失/爆炸
**解决方案**：
- Pre-norm架构应改善此问题
- 检查梯度范数：应在1-100范围内
- 调整clip_grad：尝试3.0或10.0
