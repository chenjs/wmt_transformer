# 定量评估报告：200,000步训练模型
**报告生成时间:** 2026-03-06
**评估模型:** `models/best_model.pt` (步数 207,239)
**评估方法:** 贪婪解码 (greedy) 与束搜索解码 (beam4, beam_size=4)
**测试集大小:** 40个句子，分5个难度等级
**评估脚本:** `scripts/calculate_quantitative_metrics.py` (已修复beam方法支持)

## 🔥 关键发现

### 解码策略的显著影响
**束搜索解码 (beam4) 相比贪婪解码有显著提升**:
- **BLEU分数**: **10.51 → 22.19** (+110%提升)
- **翻译编辑率 (TER)**: **1.162 → 0.772** (降低33.6%)
- **词汇重叠度**: **24.9% → 30.0%** (提升20.5%)
- **长度比**: **1.49 → 1.12** (更接近参考长度)

**结论**: 模型**具备中等翻译潜力**，但需使用适当解码策略。贪婪解码严重低估了模型能力。

- **模型潜力**: 比初步评估更高，但仍有改进空间

## 📈 详细结果分析

### BLEU分数分析
- **贪婪解码**: 10.51 (百分制) - 基本不可用
- **束搜索解码**: 22.19 - 可理解但需人工修正
- **改善领域**: 所有难度级别均有显著提升

**BLEU分数等级解读**:
- <20: 基本不可用 (贪婪解码)
- 20-30: 可理解但需人工修正 (束搜索解码达到此范围)
- 30-40: 可用质量 (下一阶段目标)
- 40-50: 良好质量
- >50: 优秀质量

### 翻译编辑率 (TER) 分析
- **贪婪解码平均TER**: 1.162 (高编辑需求)
- **束搜索解码平均TER**: 0.772 (中等编辑需求)
- **最佳表现**: 复杂句子 TER=0.2-0.4
- **最差表现**: 基础词汇 TER=1.5-2.5 (两种方法均存在)

**TER解读**:
- 0.0-0.5: 优秀
- 0.5-0.8: 良好 (束搜索达到此范围)
- 0.8-1.0: 可用
- >1.0: 需要大量编辑 (贪婪解码)

## 🎯 主要问题识别与改善

### 束搜索解决的瓶颈问题
1. **基础词汇映射**: beam4修复了Hello→Hallo, Good morning→Guten Morgen
2. **名词准确性**: sky→Himmel (非Luftraum), test→Test (非Prüfstand)
3. **输出精简**: 减少冗余表达，长度比更合理

### 仍存在的核心问题
1. **零精确匹配**: 仍无完全正确的翻译
2. **部分词汇错误**: 少数专有名词仍不准确
3. **风格不一致**: 输出有时过于正式或冗长

### 解码策略启示
- **模型潜力被低估**: 贪婪解码严重低估了模型实际能力
- **解码策略关键**: 翻译质量高度依赖解码算法
- **实用建议**: 实际部署必须使用束搜索解码
